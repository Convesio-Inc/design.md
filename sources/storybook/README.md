# Storybook Sources

Use this directory for curated current product component references.

Storybook contributes current component behavior, supported variants, and implementation-facing contracts.

Rules:
- Include only currently used product components.
- Exclude deprecated, legacy, or experimental component docs in v1.
- Capture variants that affect canonical token or component decisions.
- Keep references concise and tied to short IDs used in `design.md`.
- Include concise capture metadata (date, owner, and scope) per curated entry.

Authority and conflict rules are defined canonically in `../../design.md`.

Short ID examples:
- `storybook:button-primary`
